# Functional Requirements

## 1. User Management

**FN-01**: The system shall allow administrators to create, update, deactivate, and delete user accounts.

**FN-02**: The system shall allow users to authenticate using a secure login mechanism.

**FR-03**: The system shall allow administrators to assign roles to users.

## 2. Role Management

**FN-04**: The system shall support multiple roles (CEO, Department Head, Employee).

**FR-05**: The system shall restrict system access and actions based on user roles.

## 3. Department Management

**FN-06**: The system shall allow administrators to create, update, and delete departments.

**FR-07:** The system shall allow department heads to **view, add, edit, and delete members** within their department.

## 4. Project Management

**FN-08**: The system shall allow department heads to create, update, and close projects.

**FR-09:** The system shall allow projects to be associated with **one or more departments** .

**FN-10**: The system shall allow users to view projects based on their role and department.

## 5. Task Management

**FN-11**: The system shall allow department heads to create and assign tasks to employees.

**FN-12**: The system shall allow employees to update the status of their tasks.

**FN-13**: The system shall allow tasks to have priorities and deadlines.

## 6. Progress Tracking

**FN-14**: The system shall display task progress in real time.

**FN-15**: The system shall provide project progress indicators for management users.

## 7. Notifications

**FN-16**: The system shall notify users when tasks are assigned or updated.

## 8. Reporting

**FR-17**: The system shall provide basic reports on project and task status.
